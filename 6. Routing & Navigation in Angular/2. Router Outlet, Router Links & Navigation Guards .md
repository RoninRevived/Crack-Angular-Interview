## **ðŸ”¹ What is Angular Routing?**  
Angular **Routing** allows navigation between different views **without reloading** the entire page. It provides seamless **Single Page Application (SPA)** behavior using **Router Outlet, Router Links, and Navigation Guards**.

---

## **ðŸ”¹ Router Outlet: The Dynamic View Loader**  
The `<router-outlet>` is a **placeholder** where Angular dynamically loads components based on the route.

### **ðŸ“Œ Example: app.component.html**
```html
<nav>
  <a routerLink="/">Home</a>
  <a routerLink="/about">About</a>
</nav>

<!-- This is where the routed component will be displayed -->
<router-outlet></router-outlet>
```
âœ… **Each time you navigate, Angular replaces `<router-outlet>` with the matched component.**

---

## **ðŸ”¹ Router Links: Navigating Without Page Reload**  
Instead of `<a href>`, Angular uses `[routerLink]` to navigate without refreshing the page.

### **ðŸ“Œ Example: app.component.html**
```html
<nav>
  <a routerLink="/">Home</a>
  <a routerLink="/about" routerLinkActive="active">About</a>
</nav>
```
âœ… `routerLinkActive="active"` **adds an "active" class** to the link when the route matches.  
âœ… `routerLink="/about"` **navigates to `/about`** without reloading the page.

---

## **ðŸ”¹ Programmatic Navigation Using Router Service**  
You can navigate dynamically using the `Router` service.

### **ðŸ“Œ Example: Navigating on Button Click (home.component.ts)**
```typescript
import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-home',
  templateUrl: './home.component.html',
})
export class HomeComponent {
  constructor(private router: Router) {}

  goToAbout() {
    this.router.navigate(['/about']);
  }
}
```

### **ðŸ“Œ Example: home.component.html**
```html

```
âœ… **Now clicking the button will navigate to `/about`.**

## **ðŸ”¹ Navigation Guards: Protecting Routes**  
Navigation Guards allow us to **control access** to certain routes. They help in scenarios like:  
âœ… Restricting access to authenticated users only.  
âœ… Preventing users from leaving unsaved changes.  
âœ… Running pre-loading logic before a route loads.

---

### **ðŸ”¹ Types of Navigation Guards**
| Guard | Description |
|-------|------------|
| **CanActivate** | Checks if a user **can access** a route |
| **CanDeactivate** | Checks if a user **can leave** a route |
| **CanLoad** | Restricts loading of lazy-loaded modules |
| **Resolve** | Preloads data before loading a component |

---

## **ðŸ”¹ CanActivate: Protecting Routes**  
Prevents users from accessing a page unless they meet certain conditions (e.g., authentication).

### **ðŸ“Œ Step 1: Generate an Auth Guard**
```sh
ng generate guard auth
```
âœ… This creates `auth.guard.ts` automatically.

---

### **ðŸ“Œ Step 2: Implement `CanActivate` in `auth.guard.ts`**
```typescript
import { Injectable } from '@angular/core';
import { CanActivate, Router } from '@angular/router';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  constructor(private router: Router) {}

  canActivate(): boolean {
    const isLoggedIn = false; // Change this based on authentication logic
    if (!isLoggedIn) {
      this.router.navigate(['/login']); // Redirect to login page
      return false;
    }
    return true;
  }
}
```

âœ… **If `isLoggedIn` is false, the user is redirected to the login page.**  

---

### **ðŸ“Œ Step 3: Apply `CanActivate` to Routes**
Modify `app-routing.module.ts` to apply the guard.

```typescript

  { path: '', component: HomeComponent },
  { path: 'dashboard', component: DashboardComponent, canActivate: [AuthGuard] },
  { path: 'login', component: LoginComponent }
];
```
âœ… **Now, only logged-in users can access the `/dashboard` route.**

## **ðŸ”¹ CanDeactivate: Prevent Accidental Navigation**  
Stops users from leaving a page with **unsaved changes**.

### **ðŸ“Œ Step 1: Create `CanDeactivate` Guard**
```sh
ng generate guard unsaved-changes
```
âœ… This creates `unsaved-changes.guard.ts`.

---

### **ðŸ“Œ Step 2: Implement `CanDeactivate`**
```typescript
import { Injectable } from '@angular/core';
import { CanDeactivate } from '@angular/router';

export interface CanComponentDeactivate {
  canDeactivate: () => boolean;
}

@Injectable({
  providedIn: 'root'
})
export class UnsavedChangesGuard implements CanDeactivate<CanComponentDeactivate> {
  canDeactivate(component: CanComponentDeactivate): boolean {
    return component.canDeactivate ? component.canDeactivate() : true;
  }
}
```

---

### **ðŸ“Œ Step 3: Apply `CanDeactivate` to a Component**
```typescript
import { Component } from '@angular/core';
import { CanComponentDeactivate } from '../unsaved-changes.guard';

@Component({
  selector: 'app-edit-profile',
  templateUrl: './edit-profile.component.html',
})
export class EditProfileComponent implements CanComponentDeactivate {
  hasUnsavedChanges = true;

  canDeactivate(): boolean {
    return this.hasUnsavedChanges ? confirm('You have unsaved changes. Leave anyway?') : true;
  }
}
```

---

### **ðŸ“Œ Step 4: Apply Guard to Routes**
```typescript
const routes: Routes = [
  { path: 'edit-profile', component: EditProfileComponent, canDeactivate: [UnsavedChangesGuard] }
];
```
âœ… **Users will see a confirmation popup if they try to leave without saving.**

## **ðŸ”¹ Nested Routes (Child Routes)**
Sometimes, you need **sub-routes inside a parent route**, like:  
- `/dashboard/profile`
- `/dashboard/settings`

---

### **ðŸ“Œ Step 1: Define Child Routes**
```typescript

  {
    path: 'dashboard',
    component: DashboardComponent,
    children: [
      { path: 'profile', component: ProfileComponent },
      { path: 'settings', component: SettingsComponent }
    ]
  }
];
```
âœ… Now, `/dashboard/profile` will show **ProfileComponent** inside **DashboardComponent**.

---

### **ðŸ“Œ Step 2: Add `<router-outlet>` for Child Routes**
Inside `dashboard.component.html`:
```html
<h2>Dashboard</h2>
<nav>
  <a routerLink="profile">Profile</a>
  <a routerLink="settings">Settings</a>
</nav>

<router-outlet></router-outlet>
```
âœ… **This `<router-outlet>` dynamically loads `ProfileComponent` or `SettingsComponent` based on the route.**
